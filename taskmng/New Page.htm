Taskmng任务管理系统安装说明

　　本系统是基于B/S模式的任务管理软件，其后台数据库为SQL Server（通过对少量的数据库函数的修改，可以运行于各种类型的数据库）；其编程主要采用了ASP技术，该系统的推行一方面便于一个部门的管理工作，另一方便是为了更好地向全局推行ASP及SQLServer等新技术，以加快全局企业网建设的步伐。

  1. 系统运行环境：

    操作系统为NT4.0，(IIS3.0+ASP)/IIS4.0(自带ASP)/(SP4+ASP)(即可运行ASP)，为了对系统作最少的改动即可在您的服务器上运行，建议使用工作平台为IIS 4.0，数据库为SQL Server 6.5／SQL Server 7，程序运行的中间件为ODBC。

    (SQLServer7可在10.65.10.16上的ads/microsoft/下下载）

    安装任务管理系统前必须先得到下列文件(请使用电子邮件与王权联系wangq@dq.cnpc.com.cn,我们把它Email给你.)：

    tasksql.zip 打包后为数据库环境中所用到的tasktab.sql和dept_use.sql
    task_web.zip 打包后为WWW站点中的文件
  2. 数据库环境：

    把数据库SQL Server 正确安装到服务器上，安装成功后创建数据库taskmng（任意的库名，仅需此库名与ODBC配置中的数据库名（非数据源）相一致），打开其查询窗口，选择taskmng为当前数据库，打开文件tasktab.sql文件并执行，程序自动创建所用到的各种表，同时插入需要的环境变量，打开文件dept_use.sql文件，按照您的要求写入您单位的编码和人员信息，并执行该文件（也可自行建立测试所需的单位代码库和人员库），则把相应的数据插入对应的表中。

  3. 配置ODBC

    建立ASP程序中需要调用的数据源。在ASP所在服务器上配置ODBC的系统DSN（SystemDSN），数据源驱动程序的类型选择sql server（注：该驱动程序的版本必须与所安装的数据库的版本相一致，否则会引起不可预料的错误），（1）输入数据源名：taskview（与安装说明中提到的taskdb.txt中的dsn名相一致）；（2）输入数据库服务器的名称，此名称为安装数据库SQL Server 7的主机名，本地缺省为（local）；（3）输入数据库的名称：taskmng。

  4. 建立WWW站点

    在服务器上设立站点taskmng，把该系统所需要的文件导入到该站点（也可以在服务器站点上建立taskmng目录，再将所需文件copy到此目录下）。在NT操作系统的system32目录下创建taskdb.txt文件，在该文件中写入一行（dsn=taskview;uid=sa;pwd=），其中的DSN为ODBC中所建的数据源名称如taskview，uid为可以访问数据库的用户名，pwd为该用户的口令。

  5. 在使用前请检查系统时间的显示方式。

   （控制面板中的区域设置）

　　１）将时间中的时间样式改为HH:mm:ss

　　２）将日期中的短日期样式和长日期样式分别改成

　　　　yyyy-MM-dd和yyyy MM dd。
